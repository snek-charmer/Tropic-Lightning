# UDS bundle: keycloak-portal

Deploys the portal as part of a [UDS](https://uds.defenseunicorns.com) bundle.
The bundle composes:

1. **Zarf init** — registry + Zarf agent bootstrap.
2. **UDS Core** — Istio service mesh, Keycloak (SSO), and the UDS Operator.
3. **keycloak-portal** — this app (the Zarf package in `../zarf`).

## What UDS does for the app

When deployed via UDS, the chart renders a `uds.dev/v1alpha1` **Package** CR
(see `../helm/keycloak-portal/templates/uds-package.yaml`), and the UDS Operator
reconciles it:

- **SSO** — creates the Keycloak OIDC client and writes the generated client
  secret into the `keycloak-portal-sso` Secret. The app reads
  `OIDC_CLIENT_SECRET` from it. **You never supply the client secret.**
- **Exposure** — generates the Istio `VirtualService` on the chosen gateway
  (the chart's own VirtualService is disabled under UDS).
- **NetworkPolicies** — the namespace is default-deny; the Package declares
  egress to the **peat node** (`peat-system:50051`) and **Keycloak**. DNS,
  Istiod, and metrics policies are added automatically by the Operator.

The peat node is a **platform prerequisite** (it is not part of this bundle).

## Build

```bash
# 1) Build the app image and Zarf package (the bundle references ../zarf by path).
docker build -t keycloak-portal:0.1.0 .
zarf package create deploy/zarf --confirm --output deploy/zarf

# 2) Create the bundle (pulls init + uds-core OCI artifacts).
uds create deploy/uds --confirm
# -> uds-bundle-keycloak-portal-<arch>-0.1.0.tar.zst
```

Pin the `core` package `ref` in `uds-bundle.yaml` to the uds-core **version and
flavor** for your environment (`-upstream` | `-registry1` | `-unicorn`).

## Deploy

```bash
uds deploy uds-bundle-keycloak-portal-*.tar.zst --confirm \
  --set PORTAL_HOST="portal.example.com" \
  --set ISSUER="https://sso.example.com/realms/uds" \
  --set PEAT_NODE_ADDR="peat-node-peat-node.peat-system.svc:50051"
```

## Bundle variables (override at deploy time)

| Variable | Default | Notes |
|----------|---------|-------|
| `PORTAL_HOST` | `portal.uds.dev` | External host; also derives the OIDC redirect URLs |
| `GATEWAY` | `tenant` | UDS gateway: `tenant` / `admin` / `passthrough` |
| `ISSUER` | — | Keycloak realm issuer URL |
| `CLIENT_ID` | `keycloak-portal` | OIDC client ID (UDS creates this Keycloak client) |
| `PEAT_NODE_ADDR` | `peat-node-peat-node.peat-system.svc:50051` | peat node gRPC endpoint |
| `PEAT_NAMESPACE` | `peat-system` | peat node namespace (for the egress policy) |

The OIDC **client secret is generated by UDS** — there is no secret variable.
